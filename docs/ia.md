# Guía para IAs colaboradoras

Esta documentación está pensada para que una IA pueda:
- Entender la estructura y convenciones del proyecto.
- Saber dónde y cómo añadir código nuevo.
- Documentar sus cambios de forma útil para humanos y otras IAs.

## Reglas para la IA
- Antes de modificar, revisa el README.md del módulo.
- Si creas un nuevo feature, añade un README.md explicando propósito, modelos y flujos.
- Si modificas lógica global, actualiza `/docs/arquitectura.md` o `/docs/decisiones.md`.
- No borres documentación existente salvo que esté obsoleta y lo indiques.
- Usa comentarios claros y breves en el código.
- Si tienes dudas, documenta la decisión tomada en `/docs/decisiones.md`.
